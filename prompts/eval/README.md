# Prompt eval corpus

Golden triples used to score every prompt change. Owner in Q1: **A2 (agent-eng)**. Ownership transfers to the QA infra team in Q2 (§10 of Q1 plan).

## What's in here

| File / dir | Purpose |
|------------|---------|
| `schema.md` | The Triple schema documentation |
| `corpus/*.json` | One file per triple. Human-authored + human-reviewed. |
| `baseline.json` | Metrics from the last blessed run (generated by the harness) |
| `fixtures/` | Anonymized repo snapshots used as inputs |

## Triple schema (short version)

Each corpus file contains one `Triple`:

```json
{
  "id": "coverage.checkout-add-items.v1",
  "role": "coverage",
  "input": {
    "goal": "Add 3 items to cart and see cart total match sum",
    "targetUrl": "https://demo-fixture/products",
    "expectedOutcomes": ["cart badge shows 3", "cart total equals sum"],
    "repoFixtureId": "shop-repo-01"
  },
  "expected": {
    "testFile": {
      "path": "tests/checkout-add-3-items.spec.ts",
      "mustContain": ["cartBadge", "toHaveText('3')"],
      "mustAssertOutcomes": true,
      "styleReference": "tests/cart/remove-item.spec.ts"
    },
    "shouldNotContain": ["waitForTimeout", "page.click('body')"]
  },
  "metrics": {
    "astSimilarityTarget": 0.75,
    "styleConformanceTarget": 0.85
  },
  "tags": ["coverage", "checkout", "positive-path"]
}
```

Full field reference: [`schema.md`](./schema.md).

## Corpus composition target (Q1 W6 baseline)

- **20 synthetic** triples on a small canonical test repo (owner-controlled, no legal risk)
- **30 real** triples derived from partner repos, with anonymized fixtures under data-sharing agreement

Coverage across:
- Roles: 40 coverage, 5 onboarding, 5 judge (Q1 does not eval triage — Q2)
- Templates: login flows, listing/filter, form submission, search, checkout, admin CRUD
- Style variety: 5 different POM styles, 3 different locator preferences, both fixture-based auth and storageState auth

## Running the eval

```bash
# All triples
npm run eval --workspace=@poc/eval-harness

# Filter by prompt
npm run eval --workspace=@poc/eval-harness -- --prompt generator/system.md

# Filter by tag
npm run eval --workspace=@poc/eval-harness -- --tag coverage

# Update baseline (after a promoted prompt change)
npm run eval --workspace=@poc/eval-harness -- --write-baseline
```

Output is a Markdown report comparing current run to `baseline.json`, with per-triple pass/fail and regression flags. See [../../packages/eval-harness/README.md](../../packages/eval-harness/README.md) for the runner.

## Governance

- **Never edit `baseline.json` by hand** — only the harness writes it.
- **Never delete a triple** without documented reason. Triples get more valuable with age.
- **Anonymize before merge** — every fixture must strip customer names, PII, real domains.
- **Two-eyes review** for every new triple: A2 + one non-agent-team reviewer.
